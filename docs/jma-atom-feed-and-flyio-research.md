## Part 1: 気象庁 高頻度Atom Feed 詳細

### 【結論】

気象庁の高頻度Atom Feedは、**毎分更新・無料・登録不要**で利用でき、防災Bot開発における最も信頼性の高い情報ソースです。4カテゴリに分類された8フィード（高頻度4+長期4）から、地震・津波・気象警報・火山情報など**約70種類以上**の電文を取得可能です。

---

### 1.1 フィード全体構成

気象庁は更新頻度の異なる2種類のAtomフィードを4カテゴリずつ、計8本提供しています。

| 種別 | 更新頻度 | 保持期間 | 用途 |
|:---|:---|:---|:---|
| **高頻度フィード** | **毎分更新** | 直近10分以上 | リアルタイム監視向き |
| 長期フィード | 毎時更新 | 数日間 | 障害復旧・バックフィル用 |

### 1.2 高頻度フィード一覧（4カテゴリ）

| カテゴリ | URL | 含まれる情報 |
|:---|:---|:---|
| **定時** | `https://www.data.jma.go.jp/developer/xml/feed/regular.xml` | 天気概況など定時発表情報 |
| **随時** | `https://www.data.jma.go.jp/developer/xml/feed/extra.xml` | 警報・注意報など随時発表情報 |

### 1.4 Atomフィードのデータ構造

各フィードはRFC 4287準拠のAtom XML形式で配信されます。

```xml
<?xml version="1.0" encoding="utf-8"?>
<feed xmlns="http://www.w3.org/2005/Atom">
  <title>高頻度（地震火山）</title>
  <subtitle>JMAXML publishing feed</subtitle>
  <updated>2026-02-24T12:30:55+09:00</updated>
  <id>urn:uuid:xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx</id>
  
  <entry>
    <title>震源・震度に関する情報</title>
    <id>urn:uuid:xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx</id>
    <updated>2026-02-24T03:30:42Z</updated>
    <author><name>気象庁</name></author>
    <link href="https://www.data.jma.go.jp/developer/xml/data/xxxxx.xml" 
          type="application/xml"/>
    <content type="text">
      【震源・震度情報】24日12時27分ころ、地震がありました。
    </content>
  </entry>
  <!-- 直近10分以上のentry群 -->
</feed>
```

**重要なフィールド**:
- `entry > id` — 電文の一意ID（重複排除に使用）
- `entry > link[href]` — XML電文本体のURL（ここから詳細XMLを取得）
- `entry > title` — 情報種別名
- `entry > content` — ヘッドライン（Bluesky投稿テキストに利用可能）

### 1.5 利用上の注意事項・制限

| 項目 | 内容 |
|:---|:---|
| **ダウンロード量制限** | 1日10GB以上で**IPアドレス遮断** |
| **キャッシュ挙動** | CDN経由のため、毎分20秒頃に新データが見える（経験値） |
| **ポーリング推奨間隔** | 30秒〜1分（cronなら毎分20秒にsleepして実行が効率的） |
| **取得済みファイルの再取得** | 同一URLの再取得は帯域の無駄＆遮断リスク → ID管理必須 |
| **HSTS化** | 2022年3月よりHTTPS必須 |
| **サービス保証** | なし（メンテナンスによる停止・遅延あり） |
| **二次利用** | 自由（パブリックドメイン扱い） |

### 1.6 ⚠️ 2026年度の重要な電文体系変更

気象庁は**2026年度（令和8年度）出水期**より、防災情報の体系整理に伴いXML電文の統合・新設・廃止を予定しています。

| 変更点 | 内容 |
|:---|:---|
| **新規電文** | 「集約通報」「気象防災速報」等の新しい電文が追加 |
| **移行措置電文** | 現行電文の多くは新規電文運用開始後**2年間（2028年度まで）**並行提供 |
| **即時廃止** | 全般台風情報は新規電文運用開始時に即提供終了（移行措置なし） |
| **名称変更** | 竜巻注意情報(VPHW51) → 気象防災速報(竜巻注意)(VPHW51) |

Bot開発時は**新規電文への対応を見据えた設計**にすることを推奨します。

### 1.7 防災Botに推奨するフィードの組み合わせ

| 優先度 | フィード | 監視間隔 | 対象電文 |
|:---|:---|:---|:---|
| **必須** | `eqvol.xml` | 30秒〜1分 | 地震速報・津波警報・噴火速報 |
| **必須** | `extra.xml` | 1分 | 気象警報・竜巻注意・土砂災害・台風 |
| 任意 | `regular.xml` | 5〜10分 | 天気予報（日常的情報配信用） |
| 任意 | `other.xml` | 5〜10分 | 海上警報等（用途に応じて） |

## Part 2: Fly.io 概要とコスト分析

### 【結論】

Fly.ioは**グローバル分散型のアプリケーションホスティングプラットフォーム**で、東京リージョン(nrt)を含む世界18リージョンにデプロイ可能です。防災Botに最適な`shared-cpu-1x`+`256MB`構成なら**月額約$2（約300円）**で常時稼働が実現できます。ただし、**2026年時点では無料枠（Free Tier）は新規ユーザーには実質提供されていない**点に注意が必要です。

---

### 2.1 Fly.ioとは

Fly.ioは、Dockerコンテナをベースとしたアプリを世界中のエッジロケーションで実行できるPaaS（Platform as a Service）です。

**特徴**:
- **Firecracker VM**ベースの軽量仮想マシン（AWS Lambdaと同じ技術）
- **秒単位課金** — 使った分だけ支払い
- **東京リージョン(nrt)**あり — 日本国内向けサービスに最適
- **CLIベースのデプロイ** — `flyctl deploy`一発でデプロイ
- **Autostop/Autostart** — トラフィックがない時はマシンを自動停止（コスト削減）
- Dockerfile対応、Node.js/Python/Go/Rustなど言語不問
- Built-in WebSocket対応

### 2.4 Fly.ioが防災Botに適している理由

| 特性 | 防災Botとの相性 |

| **常時稼働可能** | cronジョブではなくプロセスとして常時ポーリング可能 |

| **秒単位課金** | 開発・テスト時のコスト最小化 |

## 【注意点・例外】

1. 気象庁フィードのPUSH型配信は**2020年9月に完全終了**。現在はPULL型のみ
2. 気象庁のXML電文は**2026年度に大幅な体系変更**が予定されており、Bot開発時は新旧電文への対応が必要
3. Fly.ioの料金は**リージョンによって異なる**。本レポートは東京(nrt)基準
4. Fly.ioのFree Trialクレジットは**使い切り型**で、補充されない
5. 緊急地震速報（EEW）はPULL型フィードでは取得不可。DMDATA.JP（有料WebSocket）が必要
